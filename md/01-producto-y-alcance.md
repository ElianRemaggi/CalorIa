# Producto y Alcance

## 1. Problema
Las personas que quieren bajar, subir o mantener su peso necesitan conocer cuántas calorías y macros consumen por día. Las apps actuales suelen tener demasiada fricción porque obligan a buscar alimentos manualmente, navegar bases gigantes y completar formularios repetitivos.

## 2. Propuesta de valor
Registrar comidas de forma rápida con dos caminos:
- foto + IA
- carga manual

La experiencia central debe comunicar:
- rapidez
- simplicidad
- corrección manual fácil
- progreso diario claro

## 3. Usuario objetivo
Público general:
- personas que quieren bajar de peso
- personas que quieren subir de peso
- personas que quieren mantener peso
- usuarios que no quieren usar apps complejas de fitness

## 4. Alcance del MVP

### Incluye
- login y registro con Google
- onboarding nutricional:
  - género
  - edad
  - altura
  - peso
  - objetivo: bajar, subir, mantener
- cálculo de calorías objetivo
- cálculo de macros objetivo
- registro de comida por foto
- registro de comida manual
- análisis IA con:
  - calorías
  - proteínas
  - carbohidratos
  - grasas
  - breve descripción de ingredientes
- edición manual de resultado
- dashboard diario
- historial diario, semanal y mensual
- recordatorios configurables
- configuración de proveedor de IA
- almacenamiento local de API key

### No incluye
- almacenamiento persistente de fotos
- iOS publicado
- pagos
- suscripciones
- panel admin
- integración con wearables
- escáner de código de barras
- coaching nutricional
- recomendaciones médicas
- redes sociales

## 5. Diferenciadores
- registrar comidas en segundos
- menor fricción que apps basadas en búsqueda manual
- soporte para varios proveedores de IA
- el usuario conserva control sobre su API key en el MVP

## 6. Objetivos de negocio de la primera etapa
- validar uso real
- conseguir usuarios
- medir retención
- entender precisión percibida
- detectar qué porcentaje de usuarios corrige manualmente la IA

## 7. Métricas prioritarias
- tasa de activación
- tasa de onboarding completado
- comidas registradas por usuario por día
- retención D1, D7, D30
- porcentaje de corrección manual del resultado IA
- tiempo promedio para registrar una comida

## 8. Reglas funcionales
- el usuario puede editar calorías y macros detectados
- el resultado final guardado debe diferenciar valor estimado y valor corregido
- el sistema nunca debe presentar el cálculo IA como exactitud clínica
- el usuario puede elegir entre foto o carga manual
- el dashboard debe mostrar consumido y restante

## 9. Riesgos del producto
- baja confianza en estimaciones
- latencia alta
- fricción en onboarding
- falta de hábito de uso
- confusión entre estimado y confirmado

## 10. Criterios de éxito del MVP
- el usuario puede registrarse con Google y completar onboarding sin errores
- el usuario puede registrar una comida por foto y otra manualmente
- el dashboard muestra calorías y macros consumidos/restantes
- el historial permite navegar por día, semana y mes
- el usuario puede configurar recordatorios y proveedor IA
